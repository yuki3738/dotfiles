# TOHOシネマズ予約サポート

「観たい映画 × 観たい劇場 × 観られる時間」を素早く決めるための対話型Skill。最終決済はユーザーに任せる。

## 役割分担

| ステップ | Claude | ユーザー |
|---------|--------|---------|
| 上映中作品の取得 | ◯ |  |
| 好みヒアリング → サジェスト | ◯ |  |
| 都内劇場の比較・提案 | ◯ |  |
| 上映時間の集計・提示 | ◯ |  |

## 前提

- ブラウザ自動化は **playwright-cli** を最優先で使う（TOHOサイトはJS必須）
  - playwrightが使えないときは `mcp__claude-in-chrome__*`（要 ToolSearch ロード）にフォールバック
- 静的に取れる範囲は `defuddle parse <url> --md` を使ってトークン節約（インストールパス: `/Users/minamiyayuki/.anyenv/envs/nodenv/versions/24.10.0/lib/node_modules/defuddle-cli/dist/cli.js` 経由でも可）
- TOHO CINEMAS 会員ログインや決済情報には触れない
- ハルシネーション禁止: 公式サイトに無い上映情報・割引情報・館の設備は書かない（[グローバルCLAUDE.md](file:///Users/minamiyayuki/.claude/CLAUDE.md) の「情報の正確性」原則）

| 用途 | URL |
|------|-----|
| 上映中作品一覧 | https://www.tohotheater.jp/movie/ |

| 都内劇場一覧 | https://www.tohotheater.jp/theater/find.html?area=tokyo |

## 手順

### ステップ1: 起動時のヒアリング

ユーザーから最低限の情報を引き出す（既に発話に含まれていれば省略）:

1. **観る日時**（例: 今晩 / 明日午後 / 今週末）
2. **エリア／最寄駅** または「どこでもOK」
3. **観たい雰囲気**（例: アクション系／泣ける系／話題作なんでも／家族向け）
4. **同行者**（一人 / カップル / 家族 / 友人）
5. **譲れない条件**（IMAX希望、字幕／吹替、上映時間2h以内 など）

3-5は箇条書きで質問するとサクサク答えてもらえる。**全部一括で聞かない、必要な分だけ。**

### ステップ2: 上映中作品の取得

```bash
# JS要らない一覧ページなら defuddle で十分
npx defuddle-cli parse https://www.tohotheater.jp/movie/ --md
```

- defuddleで取れない場合は playwright-cli の `browser_snapshot` で取得
- 作品名、公式コピー、ジャンル、上映時間（分）、レーティング、公開日、movie_cd を控える
- 「公開予定」と「上映中」を混同しない

### ステップ3: サジェスト

ユーザーの好みと作品リストを突き合わせ、**3〜5本に絞ってランキング**形式で出す。各作品について:

- タイトル（ジャンル / 上映時間 / レーティング）
- 一言サマリー（公式情報のみ。あらすじ妄想禁止）
- なぜユーザー向けか（理由を1〜2行）
- 上映が続きそうか（公開日が古いと終映リスク）

最後に「どれを観ますか？」と尋ねる。

### ステップ4: 劇場提案

選ばれた作品について:

1. 作品コード→ `https://www.tohotheater.jp/movie/search-result.html?movie_cd=...` で **その作品が上映中の劇場** を取得
2. ユーザーのエリア条件で都内劇場をフィルタ
3. 各候補館について「アクセス・設備（IMAX/Dolby/TCX等）・座席タイプ」を比較表で提示
4. **おすすめ1館 + 代替案2館** を理由付きで推奨
5. ユーザーに劇場を選んでもらう

### ステップ5: 上映時間の集計

選んだ劇場×作品の上映スケジュールを取得:

```
https://www.tohotheater.jp/theater/{劇場ID}/schedule.html
```

- playwright-cli の `browser_navigate` → `browser_snapshot` でDOMを取る
- 該当作品の上映回をすべて抽出（開始時刻、終了時刻、スクリーン、字幕/吹替、IMAX等の付帯情報、空席状況）
- ユーザー希望日時に合致する回をハイライト

出力例:

| 開始 | 終了 | スクリーン | 形式 | 空席 | 予約URL |
|------|------|-----------|------|------|---------|
| 19:30 | 21:45 | SCREEN1 | IMAX字幕 | ◎ | リンク |

### ステップ6: 予約導線の提示

ユーザーが回を選んだら:

1. 予約URL（`https://hlo.tohotheater.jp/...`）を提示
2. 「TOHO CINEMASサイトでログイン → 座席選択 → 決済」までユーザー自身で進めることを案内
3. **代理ログイン・代理決済はしない**

希望があれば座席選択ページまでブラウザを進めて、ユーザーがそのまま操作を引き継げるようにする。

### ステップ7: 記憶

ユーザーの好み（好きな監督、避けたいジャンル、よく行く劇場、上映時間の許容範囲など）が新たに分かったら、**`brain/Patterns.md` または `brain/Memories.md` に追記**する（[CLAUDE.md](file:///Users/minamiyayuki/src/github.com/yuki3738/obsidian-vault/CLAUDE.md)のメモリシステムに従う）。同じ質問を毎回しないため。

例:
- 「IMAXより通常上映で十分派」
- 「上映時間2時間以内を好む」
- 「日比谷 or 新宿が定番」

## 注意事項

- **上映情報の鮮度**: TOHOは週替わりで上映時間が変わる。当日〜数日先の確度は高いが、来週以降は変わる前提で案内
- **割引情報の捏造禁止**: TOHOウェンズデイ／レイトショー／シニア割など条件は変わる。公式に書いてあるもののみ言及
- **ChromeとPlaywright**: ユーザーがすでに開いているChromeの状態を壊さない。Playwrightの新規セッションを優先
- **席数表示**: 公式の空席記号（◎ ◯ △ ✕）の意味を勝手に翻訳しない
- **エラー対応**: スケジュールページの構造変更やJS読込失敗は十分ありうる。3回失敗したら手動で公式サイトを開いてもらう案内に切り替える（[グローバルCLAUDE.md](file:///Users/minamiyayuki/.claude/CLAUDE.md) の「3回間違えたら立ち止まる」原則）
