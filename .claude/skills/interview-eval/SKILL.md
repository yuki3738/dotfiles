# 面接評価シート生成

採用面接（カジュアル面談含む）の議事録を読み込み、評価シートのフォーマットに沿った入力文章を画面に出力する。

## 入力

`/interview-eval <候補者名 or ファイルパス>` で起動する。

- **候補者名の場合**: `Private/Interview/Casual/` 配下から候補者名を含むファイルを検索する
- **ファイルパスの場合**: 指定されたファイルを直接読み込む

Google Docsから直接読み込む場合は、ユーザーがURLを提供するか、`gog drive search` で検索してから `gog drive download --format txt` で取得する。

## 処理の流れ

### Step 1: 議事録の読み込みと理解

議事録を読み込み、以下の情報を抽出・整理する：

- 候補者の経歴・スキル・現職の状況
- 転職動機・転職の温度感・時期
- 他社選考状況
- 転職軸・希望条件（年収含む）
- 面談中の印象（コミュニケーション、技術理解度、志向性）
- 自社への関心度・フィット感

### Step 2: 評価シートの生成

以下のフォーマットに沿って出力を生成する。議事録から読み取れた情報のみを記入し、読み取れなかった項目は「（面談内で言及なし）」と明記する。推測で埋めない。

## 出力フォーマット

```
### 【0】求人応募に関するアンケートへの回答　済/未回答
https://forms.gle/hPBA4jV9pEvRrE2EA

---
### 【1】温度感

{S/A/B/Cから選択し、その判断理由を1行で添える}

温度感の基準：
- S：めちゃくちゃいい！ → スキル・志向性・カルチャーフィットが全て高い
- A：良さそう → 大きな懸念はなく、選考を進める価値がある
- B：懸念ややありだが進めたい → 気になる点はあるが、次の選考で確認したい
- C：可能性なし → 明確なミスマッチがある

### 【2】FB・ポジネガ

{議事録から読み取れるポジティブ要素とネガティブ要素（懸念点）を箇条書きで整理する}

**ポジティブ**
- ...

**ネガティブ / 懸念**
- ...

### 【3】今後の選考についてコメントあれば

{次の選考ステップへの提案。面談内容を踏まえて、選考で深掘りすべきポイントやアトラクト要素を記載する}

### 【4】転職活動について聞いていること

■転職の温度感
{議事録から読み取れた内容。例: 絶対転職/いいところがあれば/直近は考えていない}

■転職するなら時期は？
{例: 3ヶ月以内/半年以内/年単位くらいで}

■転職活動状況は？
{例: 他社も同等フェーズ/先に進行している会社がある/movが最初}

■転職理由 もしくは 先々転職するとしたらどういう場合か？
{議事録に記載された転職理由}

■他社選考状況
{具体的な社名やフェーズがあれば記載}

■転職軸は？
{候補者が重視しているポイント}

■条件的な希望は？
{現年収/希望年収/最低でも下回りたくない年収}

### 【5】その他共有事項あれば

{上記に収まらない補足情報があれば記載}
```

## 注意点

- 議事録に書かれていない情報を推測で補わない。事実ベースで記入する
- 温度感（【1】）は議事録全体の印象から判断し、判断根拠を必ず添える
- 【2】のFBは、選考担当者が次のステップを判断しやすいように、具体的なエピソードや発言を根拠として添える
- 【4】の各項目は、議事録に該当する会話があった場合のみ記入する
