# 選考案内メッセージ生成

カジュアル面談後の候補者に対して、選考プロセスを案内するメッセージを生成する。

## 入力

`/selection-message` で起動後、以下の情報をユーザーに確認する。

### 必須項目

1. **候補者名**（敬称なし）
2. **HireRooリンク**（コーディング試験のURL）
3. **日程調整リンク**（技術面接の日程調整用URL。未提供の場合はユーザーに確認する）

### 任意項目（デフォルトあり）

4. **書類提出**: 必要 or 不要（デフォルト: 必要）
5. **選考フロー**: デフォルトは「書類提出 → 技術面接の日程調整 → コーディング試験 → 技術面接 → 最終面接」。変更があればユーザーに確認する

未提供の項目はユーザーに質問して埋める。一度に全部聞かず、必須項目から順に確認する。

## メッセージ生成ルール

### トーン

- ビジネス敬語。丁寧だが堅すぎない
- 「くださいませ」程度の柔らかさ
- 絵文字は使わない

### 構成

```
{候補者名}さん

先日はカジュアル面談にお時間いただきありがとうございました。
選考へのご希望をいただきありがとうございます。

つきましては、選考のご案内をさせてください。

{選考ステップを番号付きで列挙}

ご不明点があればお気軽にご連絡ください。
引き続きよろしくお願いいたします。

南谷祐貴
```

### 選考ステップの書き方

各ステップは `**①ステップ名**` の形式で記載する。

**書類提出**（必要な場合のみ）:
```
**①書類のご提出**
履歴書・職務経歴書をご提出いただけますでしょうか。
```

**技術面接の日程調整**（必須）:
```
**②技術面接の日程調整**
下記リンクより技術面接の希望日時をご共有くださいませ。
{日程調整リンク}
```

**コーディング試験**（必須）:
```
**③コーディング試験（HireRoo）**
技術面接の2営業日前までに受験くださいませ。
{HireRooリンク}
※試験概要についても、リンク内に記載しております。
```

番号は書類提出の有無に応じて振り直す。

### 選考フローのカスタマイズ

ユーザーから追加のステップや順番の変更があった場合は、同じトーン・フォーマットで調整する。技術面接は必ず含める。

## 出力

画面にメッセージを表示する。ユーザーがコピーして使う想定。
