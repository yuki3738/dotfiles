# バクラク稟議申請スキル

バクラクの【稟議】稟議申請（発注前/購入前/契約前の稟議申請）フォームに、ユーザーが提供するインプット情報を元に入力する。

## ワークフロー

### Phase 1: 情報収集

ユーザーが提供するソースから必要情報を収集する。

**対応ソース:**
- Gmail MCP（`mcp__claude_ai_Gmail__gmail_search_messages` / `gmail_read_message`）
- Notion MCP（`mcp__claude_ai_Notion__notion-fetch`）
- Chrome MCP（ブラウザでURLを開いてページ内容を取得）
- ユーザーからの直接入力

**収集すべき情報:**
- サービス/製品名、取引先の正式名称
- 金額（ドル建ての場合は為替レート確認が必要）
- 契約期間
- 利用目的・背景
- 利用者

### Phase 2: 金額の確認

金額に関して以下をユーザーに確認する。曖昧なまま進めない。

1. **為替レート**: 外貨建ての場合、当日時点のレートをユーザーに確認する（例: 「$1=¥159 でよいですか？」）
2. **消費税**: 税込み金額で記載する必要があるため、消費税10%の加算が必要か確認する
3. **総額の計算**: 単価 × 数量 × 期間 × 為替 × (1 + 税率) の計算を明示してユーザーに確認する

### Phase 3: 申請内容の生成と提示

フォームに入力する内容をテーブル形式でユーザーに提示し、確認を得る。

### Phase 4: ブラウザでフォームに入力

Chrome MCPを使ってバクラクのフォームに実際に入力する。**申請の確定（「申請する」ボタン）は押さない。**

## フォームURL

```
https://workflow.layerx.jp/requests/new?form_id=01FY81R2ZHKCS0ENQJ2B625KBX
```

## フォームフィールドと入力手順

### 基本フィールド

| # | フィールド | 入力方法 | 備考 |
|---|---|---|---|
| 1 | ファイル | 手動アップロード（スキップ） | ユーザーに手動添付を案内 |
| 2 | 申請名 | `form_input` でテキスト入力 | 簡潔に（例: Datadog月額契約（2026年4月〜7月）） |
| 3 | 関連申請 | 必要な場合のみ | 過去の稟議に紐づく場合 |
| 4 | 支払金額 | `form_input` で数値入力 | **税込み**。数値のみ（カンマなし） |
| 5 | 納品予定日 | `form_input` で `YYYY/MM/DD` 入力 | 契約期間の最終日 |
| 6 | 支払予定日 | `form_input` で `YYYY/MM/DD` 入力 | 最終支払日 |
| 7 | 承認期限 | `form_input` で `YYYY/MM/DD` 入力 | 署名期限などに合わせる |
| 8 | 取引先 | 下記「取引先の入力」参照 | 正式名称 |
| 9 | 対応売上計上時期 | コンサルティング事業部のみ必須 | 通常は空欄 |
| 10 | 社内外飲食の有無 | 下記「ドロップダウン操作」参照 | 通常は「無」 |
| 11 | ツール利用の有無 | 下記「ドロップダウン操作」参照 | SaaS契約は「有」 |
| 12 | 取引内容 | `form_input` でテキストエリア入力 | 詳細な説明を記載 |
| 13 | 契約期間/取引期間 | `form_input` でテキスト入力 | 例: 2026年4月1日〜2026年7月31日 |
| 14 | その他 | 必要な場合のみ | |

### 取引先の入力

バクラクの取引先フィールドはコンボボックスで、選択肢にない取引先はチェックボックスを切り替えてテキスト入力する。

1. `read_page` で「選択肢にない取引先を入力」チェックボックスの ref_id を取得
2. チェックボックスをクリック（`computer` の `left_click`）
3. `read_page` で新しく表示されたテキスト入力欄の ref_id を取得
4. `form_input` で取引先名を入力

### ドロップダウン操作（社内外飲食・ツール利用）

バクラクのドロップダウンは React Select 系のコンポーネントで、`form_input` だけでは選択が確定しない。

1. `form_input` で検索テキストを入力（例: 「無」「有」）
2. ドロップダウンが開いたら `read_page` でオプションの ref_id を取得
3. オプション要素を `computer` の `left_click` でクリックして選択を確定

### ツール利用「有」の追加フィールド

ツール利用で「有」を選択すると追加フィールドが表示される。`read_page` で新しいフィールドの ref_id を取得してから入力する。

| フィールド | 入力方法 | 備考 |
|---|---|---|
| 利用するツール名 | `form_input` | サービス名 |
| 利用目的 | `form_input` | 利用の背景と目的 |
| 利用者 | `form_input` | メールアドレスまたは氏名/部署 |
| 発行期日 | `form_input` で `YYYY/MM/DD` | 申請日から3営業日以降 |
| Notionの確認 | ラジオボタンをクリック | 「確認した」を選択 |

## 取引内容の書き方

取引内容は稟議の要。承認者が判断できるよう、以下の構成で記載する。

```
{サービス名}の{契約形態}に伴う利用料です。

【背景】
現状の課題や移行の理由を簡潔に記載。

【契約内容】
- プラン: {プラン名}
- 単価と数量の内訳
- 月額合計
- 期間
- 総額（為替レート明記）

【コスト比較】（該当する場合）
- 現状: ¥XXX/月
- 変更後: ¥XXX/月
- 削減額/増加額

【備考】
- 補足事項
```

## 入力完了後

1. フォーム全体のスクリーンショットを撮影して確認
2. ユーザーに手動対応が必要な項目を案内する:
   - ファイル添付（見積書PDFなど）
   - 金額・日付の最終確認
3. 「確認する」「申請する」は**押さない**。ユーザーに委ねる。

## ブラウザ操作の手順

1. Chrome MCP の `tabs_context_mcp` でタブ情報を取得（`createIfEmpty: true`）
2. `tabs_create_mcp` で新しいタブを作成
3. `navigate` でフォームURLを開く
4. `read_page` でフィールドの ref_id を取得（`filter: "interactive"`）
5. 上から順にフィールドを入力
6. ドロップダウンは `form_input` → `read_page` → `left_click` の3ステップ
7. 入力完了後にスクリーンショットで確認

Chrome MCP のツールは deferred tools なので、使用前に `ToolSearch` で `select:mcp__claude-in-chrome__<tool_name>` を実行してロードする。
