# 候補者スクリーニング

候補者のプロフィール情報と `hr/positions/` 配下の求人票を照合し、カジュアル面談を実施すべきかを判断する。

## 入力

以下のいずれかの形式で候補者情報を受け取る：

1. **URL**: 候補者のプロフィールページURL（採用媒体、LinkedIn等）
2. **テキスト**: 履歴書・職務経歴書の内容をそのまま貼り付け

## 処理の流れ

### Step 1: 候補者情報の取得

- URLが渡された場合: ブラウザツール（mcp__claude-in-chrome）またはWebFetchで候補者プロフィールを取得する
  - ログインが必要なページの場合はmcp__claude-in-chrome を優先する
  - ページが取得できない場合はユーザーにテキスト貼り付けを依頼する
- テキストの場合: そのまま使用する

候補者の以下の情報を整理する：
- 技術スキル・使用言語/フレームワーク
- 業務経験・経歴
- 志向性・関心領域（読み取れる範囲で）
- **勤務条件の希望**（週4勤務・時短勤務・副業前提などの記載がないか必ず確認する）

マッチページ（/matches/）の場合は、US面談情報・メッセージ欄にも勤務条件に関する記載がないか確認する。プロフィール本文だけでなく、自己PR・希望条件・US面談メモなどに週4勤務・時短希望等が書かれていることがある。これらは早期にお見送り判断できる重要な情報なので見落とさないこと。

### Step 2: 求人票の読み込み

以下の優先順位で求人票を取得する：

1. `hr/positions/` 配下の全求人票（*.md、templates/とshared/は除く）を読み込む
2. `hr/positions/` が存在しない場合は、マッチページに紐づいている求人票URLにブラウザでアクセスして内容を取得する
3. どちらも取得できない場合はユーザーに求人票の提供を依頼する

各求人票から以下を抽出する：
- 必須スキル
- 歓迎スキル
- 求める人物像

### Step 3: 照合と判断

各求人ポジションについて、候補者の情報と照合する。

**判断基準:**
- 必須スキルの充足度が最重要。必須スキルの大半を満たさない場合はミスマッチと判断する
- 歓迎スキルの充足は加点要素として扱う
- 求める人物像は、経歴の方向性や記載内容から推測できる範囲で判断する（確証がない場合は「面談で確認」とする）
- カジュアル面談はカジュアルな場なので、明らかなミスマッチでない限りは「面談すべき」寄りの判断でよい

**経歴の整合性チェック（レッドフラグ検出）:**

必須スキルの充足だけでなく、以下の観点で経歴の整合性を確認する。複数該当する場合はお見送り寄りの判断とする。

- **経験年数と年収の乖離**: 経験年数が長いのに現年収・希望年収が市場水準より著しく低い場合、スキルの質や成果に疑問がある。例：エンジニア経験8年で年収300万未満は要注意
- **雇用形態の下降**: 正社員→契約社員・業務委託へのダウンは、パフォーマンスや職場適応に課題があった可能性を示唆する
- **卒業後・退職後のブランク**: 説明のつかない長期ブランク（1年以上）がある場合は注意
- **自己申告スキルの水増し感**: 多数のツール・言語を列挙しているが、実務で使用した記載があるのはごく一部の場合、スキルの深さに疑問がある
- **在籍期間に対して成果記載が薄い**: 長期間在籍しているのに具体的な成果・改善実績の記述が少ない場合、主体的に動いていなかった可能性がある
- **希望年収と求人レンジの乖離**: 希望年収が求人レンジの下限を大きく下回る場合、求人が求めるレベル感と候補者の自己評価にギャップがある可能性。単純に「年収面でミスマッチリスクが低い」とポジティブに解釈しない

これらは単独では判断材料にならないこともあるが、複数重なった場合は総合的にお見送りと判断する。出力時は「懸念点」セクションにレッドフラグとして明記すること。

**勤務地・リモートの要件（全ポジション共通）:**
- 国内フルリモートは問題なし。国内在住であれば勤務地は懸念点にならない
- 海外在住の場合は時差が許容範囲かどうかが判断基準。居住地そのものではなく時差が本質的な問題

**日本語力の要件（全ポジション共通）:**
- 全求人ポジションで「ビジネスレベル以上の日本語力」が必須。ものづくりにおけるコミュニケーションの性質上、相当流暢な日本語が話せることが前提となる
- 日本語力の記載がない・外国籍の場合は、他の条件にかかわらず**基本お見送り**と判断する
- 英語話者を受け入れるキャパがない
- 例外的に面談を推奨する場合は、その根拠（日本在住歴・日本語資格・日本語での発信実績等）をプロフィールから明示すること
- **ただし現職または直近の職歴が日系企業の場合は考慮の余地あり**。日本語力の判断材料の一つとして加味する

### Step 4: 出力

以下の形式で出力する：

```
## スクリーニング結果

### 候補者サマリー
（候補者の経歴・スキルの要約を2-3行で）

### マッチする求人ポジション

#### {ポジション名}
- **面談推奨**: Yes / No
- **理由**:
  - 必須スキル: （充足状況を簡潔に）
  - 歓迎スキル: （該当するものがあれば）
  - 人物像: （推測できる範囲で）
- **強み**: （このポジションで即戦力となれる点、市場での希少性など）
- **懸念点**: （経験の不足、未知数な領域など。面談で深掘りすべきポイントとして記載する）

### 面談で確認すべきポイント
（全ポジション共通で、候補者に聞いておきたい事項を箇条書きで）
```

- マッチする可能性があるポジションのみ表示する（上位2-3件程度）
- 明らかにミスマッチなポジションは省略する
- 全ポジションでミスマッチの場合はその旨を明確に伝える

## 注意点

- プロフィール情報は限られているため、断定的な判断は避ける。「面談で確認すべき点」として不明点を挙げるのは良い
- カジュアル面談のハードルは低めに設定する。少しでも可能性があれば面談を推奨する方向で判断する
- 候補者の個人情報は出力に必要最小限のみ含める
- **「面談で確認すべきポイント」はプロフィールに明記されている情報のみを根拠にする。プロフィールに書かれていないことを推測で確認ポイントに含めない。** 推測であることを記載する場合は「〜と思われるが確認」のように根拠が推測であることを明示すること
