# テックブログ レビュールール

記事レビュー時に以下の観点でチェックする。

---

## 1. 文体・表現

- 「です・ます」調で統一する。「である」調と混在させない
- 1文は100文字以内を目安にする。超える場合は分割を検討する
- 口語的な表現を避ける（「〜と思います」→「〜です」）
- 体言止めは箇条書き以外では避ける
- 「〜たり」は2回以上セットで使う（「〜たり、〜たりする」）
- 変化を伴わない文脈で「〜になります」を使わない（→「〜です」）
- 冗長な表現を避ける（「〜というのは」→「〜のは」、「〜するようなXX」→「〜するXX」）
- 曖昧な表現を避け、具体的に書く

### 例

```
❌ 複数の機能を一度にデプロイするようなリリースサイクルを採用していました。
✅ 複数の機能を一度にデプロイするリリースサイクルを採用していました。
```

---

## 2. 能動態・受動態

- 能動態を優先する。受動態は主語を意図的に隠す場合のみ使用する
- 同一段落内で能動態と受動態を混在させない

### 例

```
❌ 複数の設定ファイルはバッチ処理から順次実行されることで、対象システムの状態を更新していきます。
✅ バッチ処理が複数の設定ファイルを順次実行し、対象システムの状態を更新します。
```

---

## 3. 主語と述語

- 主語を省略しない。誰が・何がを明示する
- 主語と述語のねじれがないか確認する
- 「〜とは」は定義文の導入でのみ使用する

---

## 4. 助詞の使い方

- 助詞の誤用を確認する（を/が/に/で の使い分け）
- 助詞を省略しない
- 助詞と動詞の組み合わせが正しいか確認する
- 冗長な助詞表現を避ける
- 対比でない「〜が」の接続を避ける（→「〜、」や文を分ける）

---

## 5. 漢字とひらがなの使い分け

以下は原則ひらがなで書く:

| 漢字表記 | ひらがな表記 |
|----------|------------|
| 殆ど | ほとんど |
| 為 | ため |
| 事 | こと（形式名詞） |
| 所 | ところ（形式名詞） |
| 尚 | なお |
| 但し | ただし |
| 又 | また |
| 下さい | ください |
| 頂く | いただく（補助動詞） |
| 致します | いたします（補助動詞） |
| 出来る | できる |
| 様な | ような |
| 等 | など |

---

## 6. 語順・修飾

- 修飾語は被修飾語の近くに置く
- 語順は論理的な流れに沿って整理する
- 同一文中で同じ名詞・語句を繰り返さない（並列構造を除く）

---

## 7. 見出し・構成

- タイトルを「〜の話」で終わらせない
- 見出しの形式を統一する（体言止めか文形式か）
- 暗黙の前提は明記する
- 結論を先に述べる
- 「後述します」の多用を避ける
- 文と文のつながりを自然にする（接続詞の適切な使用）
- 複雑な概念は図解を検討する

---

## 8. 記号・フォーマット

- 全角スラッシュ（／）は使わない（→「/」または「・」）
- 略語は大文字で統一する（API, AWS, CI/CD など）
- 引用ブロック（`>`）は実際の引用のみに使う
- コード・コマンド・ファイル名はバッククォートで囲む

---

## 9. 正式名称・表記

以下の表記を守る:

| ❌ 誤 | ✅ 正 |
|-------|-------|
| Golang | Go |
| Spring boot | Spring Boot |
| docker | Docker |
| k8s（初出） | Kubernetes（初出時は正式名称） |
| Github | GitHub |
| Typescript | TypeScript |
| Javascript | JavaScript |
| ~~Next.js~~ | Next.js |
| node.js | Node.js |
| React.js | React |
| VsCode | VS Code / Visual Studio Code |
| MYSQL | MySQL |
| Postgres | PostgreSQL（正式文脈） |
| Chat GPT | ChatGPT |

- 株式会社movの表記:
  - 前株とする（「株式会社mov」が正、「mov株式会社」は誤）
  - プロダクト名を資金調達等の主語にしない

---

## 10. リンク・参照

- リンク切れがないか確認する（WebFetchで404チェック）
- リンク先が適切か確認する
- 「こちら」だけのリンクテキストを避ける（→ 具体的な説明を入れる）

---

## 11. その他

- 機密情報が含まれていないか確認する
- 未確定の将来計画を断定的に書いていないか確認する
- 著作権・帰属表記が必要な引用がないか確認する
- 人物写真の掲載許可について確認が必要な箇所を指摘する
- 追記がある場合は日付を明記する（例:「(2026/4/8追記)」）
