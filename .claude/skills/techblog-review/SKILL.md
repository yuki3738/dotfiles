# テックブログ記事レビュー

指定されたMarkdown記事を `rules.md` のルールに基づいてレビューします。

## 手順

1. 引数で指定された記事ファイルを読む
2. 同ディレクトリの `rules.md` のルールに基づいてレビューする
3. 記事内のリンクをWebFetchで確認し、リンク切れがないかチェックする
4. 問題がある場合のみ、以下のルールで出力する（「問題なし」「確認のみ」といった項目は記載しない）

## 出力ルール

**L{行番号}** - {観点}：{修正内容の要約}
```diff
- {修正前の文}
+ {修正後の文}
```

## 出力例

**L73** - 文法：「〜なこと」→「〜こと」
```diff
- ドキュメントが古いなことが原因で
+ ドキュメントが古いことが原因で
```

**L89** - 冗長表現：「〜というのは」→「〜のは」
```diff
- 自動化できるというのは大きな利点です。
+ 自動化できるのは大きな利点です。
```

## 注意事項

- レビュー結果は問題点のみ出力する。OKな箇所は出力しない
- 修正提案はdiff形式で具体的に示す
- リンク切れがあった場合は対象URLと行番号を明記する
