# Prompt Refiner

ユーザーが書いたプロンプト（AIへの指示文）を、AIがより正確に意図を理解し高品質な出力を返せるように改善する。

## 引数

$ARGUMENTS を以下のように解釈する:

- 引数あり: そのテキスト全体を改善対象のプロンプトとして扱う
- 引数なし: ユーザーに改善したいプロンプトの入力を求める

## 手順

### Step 1: プロンプトの受け取り

- $ARGUMENTS が空の場合、ユーザーに「改善したいプロンプトを貼り付けてください」と伝える
- 受け取ったプロンプトを `$ORIGINAL_PROMPT` とする

### Step 2: 分析

`$ORIGINAL_PROMPT` を以下の観点で分析する:

1. **曖昧さ**: 解釈が分かれる表現がないか
2. **構造**: 指示の順序・グルーピングは論理的か
3. **コンテキスト不足**: AIが正しく応答するために必要な前提情報が欠けていないか
4. **出力仕様**: 期待するフォーマット・長さ・トーンが明示されているか
5. **制約条件**: やってほしくないこと・避けるべきことが明記されているか
6. **ペルソナ/役割**: AIに期待する役割が定義されているか

### Step 3: 改善案の提示

以下のフォーマットで改善結果を提示する:

```
## 分析

| 観点 | 評価 | コメント |
|------|------|----------|
| 曖昧さ | ⚠️/✅ | ... |
| 構造 | ⚠️/✅ | ... |
| コンテキスト | ⚠️/✅ | ... |
| 出力仕様 | ⚠️/✅ | ... |
| 制約条件 | ⚠️/✅ | ... |
| ペルソナ | ⚠️/✅ | ... |

## 改善後プロンプト

[改善されたプロンプト全文]

## 主な変更点

- [変更1]
- [変更2]
- ...
```

### 改善の原則

- **元の意図を変えない**: ユーザーがやりたいことの本質は保持する
- **過剰に長くしない**: 必要十分な情報量に留める。冗長な説明は避ける
- **構造化する**: マークダウンの見出し・箇条書き・番号付きリストを活用する
- **具体的にする**: 「良い感じに」→「箇条書き3-5項目で、各項目は1文以内」のように具体化
- **出力例を含める**: 必要に応じて期待する出力の例を追加する
- **日本語プロンプトは日本語のまま改善する**: 言語は変えない
- **用途に応じた最適化**: ChatGPT/Claude/その他、どのAIで使うかを考慮する（不明なら汎用的に）

### Step 4: フォローアップ

改善案提示後、以下を確認する:

- 「この方向性で良いですか？調整したい点はありますか？」
- ユーザーからフィードバックがあれば反映して再提示する

## 注意事項

- プロンプトの中身が機密情報を含む可能性があるため、外部ツール（WebFetch等）には送信しない
- ユーザーが「もっとシンプルに」と言ったら簡潔版も提示する
- 改善後のプロンプトはコードブロックで囲み、コピーしやすくする
